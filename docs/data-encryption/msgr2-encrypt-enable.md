আপনার ৩-নোড cephadm ক্লাস্টারে msgr2 এর মাধ্যমে ইন-ট্রানজিট এনক্রিপশন (Encryption) চালু করা বেশ সহজ, কারণ cephadm ডিফল্টভাবেই msgr2 প্রোটোকল ব্যবহার করে।

নিচের ধাপগুলো অনুসরণ করে এনক্রিপশন নিশ্চিত করুন:

## ১. এনক্রিপশন মোড 'secure' সেট করা

Cephadm ক্লাস্টারে এনক্রিপশন কার্যকর করতে `ms_` মোডগুলোকে `secure` মোডে সেট করতে হবে। নিচের কমান্ডগুলো আপনার বুটস্ট্র্যাপ নোড থেকে রান করুন:

```bash
# সব ধরনের কমিউনিকেশনের জন্য সিকিউর মোড সেট করুন
ceph config set global ms_cluster_mode secure
ceph config set global ms_service_mode secure
ceph config set global ms_client_mode secure
```

## ২. কনফিগারেশন পরিবর্তন চেক করা

কমান্ডগুলো ঠিকমতো অ্যাপ্লাই হয়েছে কি না তা দেখতে নিচের কমান্ডটি দিন:

```bash
ceph config dump | grep ms_
```

এখানে `ms_cluster_mode`, `ms_service_mode`, এবং `ms_client_mode` এর ভ্যালু secure দেখালে বুঝবেন এনক্রিপশন এনাবল হয়েছে।

## ৩. সার্ভিস রিস্টার্ট (প্রয়োজন হলে)

সাধারণত `ceph config set` কমান্ড দেওয়ার পর ড্যামনগুলো অটোমেটিক নতুন কনফিগারেশন গ্রহণ করে। তবে নিশ্চিত হতে আপনি OSD বা অন্যান্য সার্ভিস রিস্টার্ট দিতে পারেন:

```bash
ceph orch restart osd
```

## ৪. ভেরিফিকেশন (সবকিছু এনক্রিপ্টেড কি না)

আপনার ক্লাস্টার এখন এনক্রিপ্টেড পোর্টে (v2) কথা বলছে কি না তা চেক করতে:

```bash
ceph mon dump
```

এখানে যদি দেখেন অ্যাড্রেসগুলো `v2:192.168.x.x:3300/0` ফরম্যাটে আছে, তবে বুঝবেন Messenger v2 কাজ করছে।

## ৫. পুরাতন প্রোটোকল (v1) বন্ধ করা (ঐচ্ছিক কিন্তু নিরাপদ)

যদি আপনি চান যে কোনো আন-এনক্রিপ্টেড (port 6789) কানেকশন অ্যালাউ করবেন না, তবে v1 বন্ধ করে দিতে পারেন:

```bash
ceph config set global ms_bind_msgr1 false
```

সতর্কতা: যদি আপনি CephFS বা RBD কার্নেল ক্লায়েন্ট ব্যবহার করেন, তবে নিশ্চিত হয়ে নিন আপনার ক্লায়েন্ট মেশিনের লিনাক্স কার্নেল 5.11 বা তার বেশি। এর নিচের ভার্সনগুলো msgr2 এনক্রিপশন সাপোর্ট করে না।

আপনি কি এই ক্লাস্টারে কোনো External Client (যেমন: Proxmox বা Kubernetes) কানেক্ট করতে চাচ্ছেন?
