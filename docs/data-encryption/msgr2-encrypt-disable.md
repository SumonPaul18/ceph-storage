
## ১. এনক্রিপশন মোড পরিবর্তন (Secure থেকে CRC-তে ব্যাক করা)

এনক্রিপশন বন্ধ করতে আমাদের `secure` মোড পরিবর্তন করে `crc` বা `prefer-crc` মোডে নিয়ে আসতে হবে। এতে ডেটা এনক্রিপ্ট হবে না, শুধুমাত্র ডেটা ইন্টিগ্রিটি চেক (Checksum) হবে।

আপনার বুটস্ট্র্যাপ নোড থেকে এই কমান্ডগুলো দিন:

```bash
# ক্লাস্টার, সার্ভিস এবং ক্লায়েন্ট মোডকে 'crc' মোডে সেট করুন
ceph config set global ms_cluster_mode crc
ceph config set global ms_service_mode crc
ceph config set global ms_client_mode crc
```

_টিপস: আপনি যদি চান যে ক্লাস্টার এনক্রিপশন এবং নন-এনক্রিপশন দুটোই সাপোর্ট করুক, তবে `crc secure` ভ্যালু সেট করতে পারেন।_

## ২. v1 প্রোটোকল পুনরায় চালু করা (প্রয়োজন হলে)

আপনি যদি আগের মতো পোর্ট `6789` ব্যবহার করতে চান বা পুরাতন ক্লায়েন্ট সাপোর্ট করতে চান, তবে v1 বাইন্ডিং এনাবল করুন:

```bash
ceph config set global ms_bind_msgr1 true
```

## ৩. কনফিগারেশন ভেরিফিকেশন

পরিবর্তনগুলো ঠিকমতো সেভ হয়েছে কি না চেক করুন:

```bash
ceph config dump | grep ms_
```

এখানে নিশ্চিত হোন যে ভ্যালুগুলো এখন `crc` দেখাচ্ছে।

## ৪. সার্ভিস রিস্টার্ট (অরকেস্ট্রেটর ব্যবহার করে)

নতুন (নন-এনক্রিপ্টেড) কনফিগারেশন কার্যকর করতে সার্ভিসগুলো রিস্টার্ট দেওয়া জরুরি। যেহেতু আপনি `cephadm` ব্যবহার করছেন:

```bash
# মনিটর রিস্টার্ট
ceph orch restart mon

# ওএসডি রিস্টার্ট
ceph orch restart osd.all-hosts
```

## ৫. মনিটর ম্যাপে v1 পোর্ট ফিরিয়ে আনা (ঐচ্ছিক)

আপনি যদি আগে ম্যানুয়ালি মনিটর ম্যাপ থেকে v1 সরিয়ে দিয়ে থাকেন এবং এখন আবার v1 পোর্ট (6789) ব্যবহার করতে চান, তবে অ্যাড্রেসগুলো এভাবে আপডেট করতে হবে:

```bash
# আপনার আইপি অনুযায়ী কমান্ডগুলো দিন
ceph mon set-addrs ceph1 [v2:192.168.68.248:3300/0,v1:192.168.68.248:6789/0]
ceph mon set-addrs ceph2 [v2:192.168.68.249:3300/0,v1:192.168.68.249:6789/0]
ceph mon set-addrs ceph3 [v2:192.168.68.250:3300/0,v1:192.168.68.250:6789/0]
```

## ৬. নেটওয়ার্ক ভেরিফিকেশন

চেক করে দেখুন পোর্টগুলো লিসেন করছে কি না:

```bash
# মনিটর পোর্ট চেক (৩৩০০ এবং ৬৭৮৯ উভয়ই থাকতে পারে)
ss -tlnp | grep -E '3300|6789'
```

## কেন এনক্রিপশন ডিজেবল করবেন? (ব্যবহারিক প্রেক্ষাপট)

বাস্তব জীবনে সাধারণত দুটি কারণে এটি করা হয়:

- Performance: এনক্রিপশনের কারণে CPU ওভারহেড বেড়ে গেলে বা থ্রুপুট কমে গেলে।
- Legacy Support: আপনার ক্লাস্টারে যদি এমন কোনো পুরাতন ক্লায়েন্ট (যেমন: পুরাতন কার্নেল ওয়ালা সার্ভার) যুক্ত করতে হয় যা msgr2 বা AES এনক্রিপশন বোঝে না।

সতর্কতা: পাবলিক নেটওয়ার্কে Ceph চালালে এনক্রিপশন ডিজেবল করা অত্যন্ত ঝুঁকিপূর্ণ।

আপনি কি পারফরম্যান্স ইস্যুর কারণে এনক্রিপশন বন্ধ করছেন, নাকি অন্য কোনো কারিগরি সমস্যায় পড়েছেন?
